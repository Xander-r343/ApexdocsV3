The Heading Tuner calibrates the **heading (rotation) controller** — the PID/PDFL loop responsible for maintaining and correcting the robot's angular orientation. Proper tuning ensures precise rotation to target headings with minimal overshoot and drift.

<Callout type="warning">
  Clear a large open area before running this tuner. The robot will rotate up to 180 degrees in place.
</Callout>

Select the **Heading Tuner** OpMode on your Driver Station and follow the on-screen telemetry prompts to begin.

---

### Controls

| Button | Action |
|--------|--------|
| `gamepad1.x` (hold) | Rotate to 180° |
| `gamepad1.b` (hold) | Rotate to −45° (315°) |
| `gamepad1.a` (hold) | Return to 0° (origin heading) |

---

### Tuning Procedure

- **Hold X** to rotate the robot to 180° and observe the turn response.
- **Hold B** to rotate the robot to −45° and check for overshoot or oscillation.
- **Hold A** to return the robot to its 0° origin heading.
- Observe the robot's angular behavior during each rotation — look for oscillation, heading drift, or sluggish correction.
- Adjust the following parameters in **Panels** based on observed behavior:
  - **Proportional Gain (P)** — Controls how aggressively the robot corrects heading error. Increase to tighten angular tracking; too high causes oscillation.
  - **Derivative Gain (D)** — Dampens rotational correction. Increase to reduce overshoot when settling on a target heading.
  - **Minimum Power (F)** — Sets a feedforward floor to overcome static friction during small angular corrections.
  - **Deadzone (L)** — Defines the angular error threshold below which no correction is applied. Reduces jitter when holding a heading.

---

### Applying Your Values

Once tuning is complete, update the following line in your robot configuration with the values obtained:

```java
.setHeadingCoeffs(new PDFLCoefficients(p, d, f, l))
```

<Callout type="info">
  Replace `p`, `d`, `f`, and `l` with the final tuned values shown in telemetry.
</Callout>
